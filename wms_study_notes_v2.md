## 一、3PL 基础数据

### 1.1 仓库管理 (Warehouse Management)

**核心功能**:
- 仓库新增、编辑、删除、启用、禁用
- 支持仓库层级管理（父仓库-子仓库）

**关键字段**:
| 字段 | 规则 |
|------|------|
| 仓库编码 | 系统自动生成，4位字符，同仓库服务商下唯一 |
| 仓库名称 | 最多30字符，同服务商下唯一 |
| 面积 | 必填，保留2位小数，单位：平方米 |
| 联系人/电话/邮箱 | 标准验证规则 |

**业务规则**:
- 父仓库禁用前必须先禁用所有子仓库
- 状态流转: 创建 → 启用/禁用
- 启用后自动同步到4PL平台

---

### 1.2 货主货品管理 (Shipper's Goods Management)

**数据来源**: 从上游ERP实时同步

**同步规则**:
1. 先验证货主是否存在且已启用
2. 货主+外部商品编码组合判断是否已存在
3. 新品生成规则: 货主编码 + 7位流水号(0000001起)
4. 已有商品更新时受限制:
   - 仓库实物库存必须为0
   - 无待处理单据（入库通知、收货差异、销退、盘点）

**商品层级**:
- 支持1-3级包装单位
- 每级含: 长/宽/高/重量/体积
- 整散转换关系

**辅助管理字段**:
- 保质期管理 (Yes/No)
- 批次号管理 (Yes/No)
- 剩余保质期(天)
- 剩余锁定期(天)

---

### 1.3 店铺管理 (Store Management)

**业务定义**: 个人或企业在电商平台注册的销售店铺

**关键字段**:
| 字段 | 规则 |
|------|------|
| 店铺编码 | 同货主下唯一，最多30字符 |
| 店铺名称 | 同电商平台下唯一，最多50字符 |
| 电商平台 | 数据字典: 天猫/京东/拼多多/淘宝/抖音/唯品会等 |
| 商家编码 | 部分平台(顺丰/快手)取面单需要 |
| 是否主店铺 | 一个货主可有多个店铺，区分主次 |
| 店铺模式 | 唯品会特有: MP/非MP |

**电商平台列表**:
- TM(天猫)、JD(京东)、PDD(拼多多)、TB(淘宝)
- YZ(有赞)、DY(抖音)、WPH(唯品会)、KS(快手)
- XHS(小红书)、WXSPH(微信视频号)

## 二、WMS 基础数据

### 2.1 箱型管理 (Box Management)

**业务定义**: 发货前的包装材料规格管理

**核心字段**:
| 字段 | 规则 |
|------|------|
| 箱型编码 | 同仓库唯一，最多15字符 |
| 箱型名称 | 同仓库唯一，最多30字符 |
| 长/宽/高 | 必填，正整数，最大5000mm |
| 体积 | 自动计算(长×宽×高)，保留4位小数 |
| 重量 | 必填，最大100kg，3位小数 |
| 分组号 | 用于推荐箱型，最多6位 |
| 预留系数 | 箱型推荐时使用，表示不可用体积比例 |

**特殊标识**:
- 是否唯品会专用箱
- 是否淘宝专用箱
- 货主专属配置（不选=全仓库可用）

**状态规则**:
- 创建: 除仓库外均可修改
- 禁用/启用: 仓库、货主、箱型编码不可改

---

### 2.2 库区管理 (Reservoir Area Management)

**库区类型**:
- 货品区、容器区、工具区、耗材区

**关键属性**:
| 属性 | 说明 |
|------|------|
| 用途属性 | 存货、拣货、收货、退货、中转 |
| 流转属性 | 库存、暂存 |
| 补货属性 | 可配置 |
| 退货属性 | 可配置 |
| 货主属性 | 单货主/多货主 |
| 货位属性 | 一品一位/一位多品 |
| 排属性 | 单排/多排 |
| 是否贵重品 | Yes/No |
| 温控属性 | 按数据字典 |
| 存储规格 | 按数据字典 |
| ABC分类 | A/B/C |
| 楼层 | 整数，同仓库唯一，最大999999 |

**用途与流转联动规则**:
- 用途=存货 → 流转只能是库存
- 用途=拣货/收货/退货/中转 → 流转可选库存/暂存
- 其他用途 → 流转只能是暂存

---

### 2.3 货位管理 (Cargo Position Management)

**货位编码规则**:
- 组成部分: 巷道(tunnel)/排(row)/组(group)/层(layer)/位(bit)
- 每部分最多3字符，字母或数字组合
- 多位组合时首位不能为0
- 分隔符"-"只加在最后选中项后

**示例**:
- 仅选"位" → 编码: "11"
- 选"排-层-位" → 编码: "B01-01-11"

**货位状态**:
- 空闲、未满、已满

**物理属性**:
| 字段 | 规则 |
|------|------|
| 长/宽/高 | 必填，2位小数，最大999 |
| 体积 | 自动计算，4位小数 |
| 最大体积 | 必填，2位小数 |
| 最大承重 | 必填，2位小数 |
| 可用体积比 | 0-1之间，2位小数 |
| 可用重量比 | 0-1之间，2位小数 |

**批量生成规则**:
- 按范围生成（如巷道01-03）
- 自动递增避免重复
- 计算生成数量

**状态修改规则**:
- 实物库存>0时不可禁用

---

### 2.4 容器类型/容器管理 (Container)

**容器分类**:
- 容器_托盘、容器_周转箱
- 工具_拣货车、工具_播种墙、工具_集货小车

**容器类型字段**:
| 字段 | 规则 |
|------|------|
| 长/宽/高 | 2位小数，最大999 |
| 体积 | 自动计算 |
| 最大承重 | 2位小数，最大99999 |
| 网格 | 可选，勾选后显示行列 |
| 列/行 | 正整数，最大99 |
| 网格排序规则 | 数字/字母/数字 |

**容器管理**:
- 容器编码同仓库服务商下全局唯一
- 状态: 创建/启用/禁用/占用
- 占用状态下不可修改/禁用

---

### 2.5 月台管理 (Platform Management)

**核心字段**:
| 字段 | 规则 |
|------|------|
| 月台编码 | 系统自动生成，10字符，同仓库唯一 |
| 月台名称 | 30字符，同仓库唯一 |
| 月台吞吐量 | 正整数 |
| 是否共用 | Yes/No |
| 是否提供升降台 | Yes/No |
| 停靠上限 | 正整数 |
| 使用顺序 | 正整数，同仓库唯一 |

**状态控制**:
- 占用状态下不可修改/禁用
- 支持释放月台（占用数清零）

---

### 2.6 装卸队管理

**关键字段**:
| 字段 | 规则 |
|------|------|
| 装卸队编码 | 系统自动生成，10字符 |
| 装卸队名称 | 30字符，同仓库唯一 |
| 服务提供方 | 业务类型为"劳务"的启用状态群组 |
| 负责人/电话 | 标准验证 |
| 成员 | 逗号分隔，最多500字符 |

---

### 2.7 仓库分类管理

**树形结构**:
- 支持多级分类
- 新增项默认显示在同级最上方
- 编码系统生成，不可重复
- 名称30字符，同仓库唯一

**导入规则**:
- 父分类不存在则导入失败
- 编码+名称相同则更新

---

### 2.8 工作台管理

**工作台类型**:
- 收货质检、销退质检、订单质检
- 订单播种、订单称重
- 退供质检、新品称重

**绑定规则**:
- 可绑定MAC地址
- 绑定状态下不可禁用

---

### 2.9 货主供应商

**数据来源**: 上游系统采购入库通知单

**解析字段**:
- 供应商ID、供应商名称（至少一个有效）
- 联系人、手机、电话（可选）

**更新规则**:
- 供应商ID不为空: ID不存在则新增，存在则更新
- 供应商ID为空: 名称不存在则新增，存在则更新
- 操作人记录为"系统"

---

### 2.10 货品管理

**编辑限制**:
- 长/宽/高/重量可编辑性由【规则配置】控制

**包装单位配置**:
| 级别 | 可选单位 |
|------|---------|
| 一级 | 整箱、散件、异形件 |
| 二级 | 散件、异形件 |
| 三级 | 散件、异形件 |

**关联货位**:
- 可选拣货库位
- 可选存储库位

**库区拣货设置**:
- 参与拣货时显示两个复选框
- 可选"按批次拣货"、"按生产日期拣货"
- 可选设置拣货数量阈值

**图片管理**:
- 1张主图（必填）
- 多张辅图（可选）
- 支持批量上传、编辑图片名称

---

### 2.11 货品存储规则

**规则目的**: 设置货主+货品+库区的补货上下限

**关键字段**:
| 字段 | 规则 |
|------|------|
| 货主 | 必填，当前仓库已授权货主 |
| 商品编码 | 必填，20字符，验证是否属于该货主且已启用 |
| 库区 | 必填，货主已分配的拣货区 |
| 补货上限 | 正整数，≥补货下限 |
| 补货下限 | 正整数，≤补货上限 |

**状态规则**:
- 创建: 除仓库外均可修改
- 禁用: 仓库、货主不可改
- 启用: 仓库、货主、商品编码、库区不可改

**重复检查**:
- 仓库+货主+库区+商品编码组合不可重复（含创建/禁用/启用状态）

## 三、通用规则总结

### 3.1 状态流转
```
创建 → 启用
  ↓
禁用（可重新启用）
```

### 3.2 删除限制
- 只有"创建"状态的记录可删除
- 删除前需确认提示

### 3.3 修改限制
- 编码类字段通常不可修改
- 启用状态下基础关联字段不可改

### 3.4 数据验证
- 所有手工输入自动去除前后空格
- 编码类：字母数字组合，长度限制
- 数值类：正整数/正数，小数位限制
- 必填项：光标定位+提示"{字段名}不能为空"

### 3.5 导入导出
- 导入自动去空格
- 失败行单独说明原因
- 编码不存在则新增，存在则更新

### 3.6 操作日志
- 所有操作（含新增）记录日志
- 按时间倒序排列
- 包含：操作时间、内容、账号、人员

## 四、关键业务联动

| 模块 | 联动逻辑 |
|------|---------|
| 货主货品 | 从3PL同步，受库存和单据状态限制更新 |
| 店铺 | 从上游导入，绑定电商平台 |
| 箱型 | 按货主/仓库维度配置，用于发货推荐 |
| 库区 | 用途与流转属性联动控制 |
| 货位 | 编码规则驱动，支持批量生成 |
| 容器 | 占用状态控制修改/禁用 |
| 供应商 | 采购入库通知驱动新增/更新 |
| 货品存储规则 | 驱动补货任务生成 |
